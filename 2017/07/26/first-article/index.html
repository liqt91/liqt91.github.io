本文记录在macos上部署hexo，同步到github pages博客的过程。

github pages 是github提供的一个静态网站服务，用以展示信息。很多人用它来搭建博客。在注册之后，它提供一个默认的域名<github用户名>.github.io以供访问。注册的方法很简单，只需要先注册一个github账户，然后在github中新建一个仓库名称为.github.io的仓库即可。供访问的静态html页面将存放在这个仓库中，我们将通过hexo这个工具生成静态网页并同步到这个仓库中，稍后会讲到。ps：二级域名必须为。

##先介绍一下hexo
除了刚开始用作测试的index.thml之外，手动建立静态的html页面显然不实际。较为流行的方法是使用hexo来生成网站。生成的网站将全部由静态页面构成，适合用作个人博客。
此外也可以用其他的方法建站，然后转换为静态页面。例如对于flask建立的网站，可以用flask-frozen插件转换为静态网站。这样将失去flask的后台，没有体现flask的优势，功能受限操作繁琐，而且跟hexo建立的静态网站没有太大差别，相比之下还是hexo更加合适。
hexo是一个快速简洁高效的博客框架。它提供一个博客所需的基本功能，也可以通过插件的方式扩展功能。它的博文支持markdown（或者其他渲染引擎）格式，因此你可以在本地编辑好markdown格式的文本之后通过简单的一行命令将博文发布到你的博客上去。虽然你的网站都是由静态页面构成的，但是hexo会帮你处理好所有链接关系。最后，hexo基于node.js，使用git发布，所以如果要使用的话需要先安装node.js和git。

##node.js 和 git的安装

  • node.js的安装很简单，只需要到官网下载对应的版本，一路确定即可。如果安装过brew，那么可以通过命令行的方式来安装nodejs：
    brew install node
  • git的安装也很简单，只需要去官网下载对应版本安装即可。之后再配置一下用户名、邮箱和ssh，教程很多。

##hexo安装
安装好node.js 和git 之后，就可以通过npm这个nodejs的包管理工具安装hexo了。
需要说明的是，npm的源因为某些原因访问速度很不理想，建议改用淘宝提供的cnpm分流作为包管理工具，cnpm的安装方式为：

npm install -g cnpm --registry=https://registry.npm.taobao.org

之后在需要使用npm的地方用cnpm代替就可以了。
之后在本地建立一个目录，用作博客的根目录。博客的所有内容将会保存在这个根目录下，我们的操作也会在这个博客的根目录下执行。
cd 到博客根目录，执行npm install hexo安装hexo，再执行hexo init初始化当前目录为hexo根目录（将会在当前目录下生成若干个hexo相关文件）

也可以不进入根目录，使用hexo init <floder_path>来初始化对应的目录。

安装完毕。

##hexo部署
之后可以使用hexo generate来生成静态页面，生成的页面可以在hexo\public\<日期>目录下找到。
使用hexo server即可启动服务，访问http://localhost:4000 供本地预览。
如果需要在githubpages上显示，需要先安装一个hexo-git工具

npm install hexo-deployer-git --save

需要注意的是，type, repo, branch冒号后面都有一个空格，不能遗漏，否则配置无效。
最后执行一下hexo deploy即可，访问.github.io试试～
